# 智能食谱助手架构设计文档

## 架构概览

项目采用 Next.js App Router。前端负责 Chat 交互、SSE 消费、结构化卡片和工具轨迹展示；后端 API Route 负责模型调用、工具调用循环、公开 API 查询和错误处理。

```txt
用户
  ↓
app/page.js
  ↓
POST /api/chat
  ↓
app/api/chat/route.js
  ↓
lib/agent.js
  ↓
OpenRouter Chat Completions API / OpenAI Responses API
  ↓
lib/tools.js
  ↓
TheMealDB / TheCocktailDB
```

## 技术栈

- Next.js
- React
- OpenAI Node SDK
- OpenRouter API
- TheMealDB
- TheCocktailDB
- CSS

## 模块设计

### `app/page.js`

负责前端交互：

- 维护聊天消息列表。
- 发送 `POST /api/chat` 请求。
- 解析 SSE 事件。
- 渲染 Markdown 回复。
- 渲染菜谱 / 饮品卡片。
- 渲染工具调用详情。
- 维护多轮对话上下文和当前卡片上下文。

### `app/api/chat/route.js`

负责服务端接口：

- 接收 `messages` 和 `cardContext`。
- 校验用户输入。
- 调用 `runFoodAgentStream`。
- 通过 SSE 返回 `trace`、`cards`、`delta`、`done`、`error`。
- 统一转换用户可读错误提示。

### `lib/agent.js`

负责 Agent 运行时：

- 读取 `.env.local` / `.env` / `process.env`。
- 选择 OpenRouter 或 OpenAI。
- 维护系统提示词。
- 规范化多轮消息。
- 执行模型工具调用循环。
- 将工具结果回传模型。
- 对食谱意图强制工具查询。

模型路径：

- OpenRouter：Chat Completions API + tools。
- OpenAI：Responses API + tools。

### `lib/tools.js`

负责工具定义和工具实现：

- `toolDefinitions`：暴露给模型的工具 schema。
- `toolHandlers`：服务端实际执行逻辑。
- `fetchJson`：公开 API 请求、超时和错误处理。
- 数据归一化：把 TheMealDB / TheCocktailDB 原始数据转换为前端卡片结构。

## Tool 设计

### `search_meals`

输入：

- `query`
- `mode`: `name` / `ingredient` / `category` / `area` / `random`
- `limit`

输出：

- `meals`
- `cards`

### `get_meal_detail`

输入：

- `id`

输出：

- `meal`
- `cards`

### `search_cocktails`

输入：

- `query`
- `mode`: `name` / `ingredient` / `random`
- `limit`

输出：

- `cocktails`
- `cards`

## SSE 事件

- `trace`：单次工具调用记录。
- `cards`：结构化菜谱 / 饮品卡片。
- `delta`：模型回复文本增量。
- `done`：响应完成。
- `error`：服务端错误。

## 配置设计

OpenRouter 配置：

```txt
OPENROUTER_API_KEY
OPENROUTER_MODEL
OPENROUTER_BASE_URL
```

OpenAI 配置：

```txt
OPENAI_API_KEY
OPENAI_MODEL
OPENAI_BASE_URL
```

选择规则：

```txt
存在 OPENROUTER_API_KEY 或 baseURL 包含 openrouter.ai → OpenRouter
否则 → OpenAI
```

## 错误处理

- 缺少模型 API Key：返回配置提示。
- 模型鉴权失败：返回鉴权提示。
- 公开 API 超时或失败：返回菜谱数据暂不可用。
- 工具参数解析失败：返回工具参数异常。
- SSE 解析失败：前端展示响应解析失败。
